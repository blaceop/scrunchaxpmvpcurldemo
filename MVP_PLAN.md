# Scrunch AXP MVP - 项目计划说明

## 1. 项目背景与核心问题

传统的企业网站和内容系统主要是为**人类用户**设计的，其特点是视觉化、交互性强，信息通常以非结构化的文本、图片、表格等形式呈现。然而，当今的AI智能体（如LLMs）在解析和利用这些“人类友好”的内容时，面临巨大挑战：它们难以准确提取语义、理解上下文，从而无法高效地实现自动化调用和知识问答。

**Scrunch AXP (Agent Experience Platform)** 的核心目标是解决这一问题，通过构建一个**“人类友好 + AI智能体友好”的双轨内容体系**。这意味着：

*   **人类用户**仍然能够享受到现有网站的完整视觉和交互体验。
*   **AI智能体/LLM**能够从同一个内容源获取到高度结构化、语义化、机器可读的内容，从而实现高效解析和调用。

## 2. MVP目标

本MVP（最小可行产品）旨在以最简洁的方式验证 Scrunch AXP 的核心概念：**在不改变外部访问URL的情况下，根据访问者的身份，动态提供不同格式的内容。**

我们将通过一个具体的“产品详情页”案例来演示这一能力。

## 3. MVP技术选型与实现

为了快速验证核心概念，MVP将采用以下技术和实现路径：

*   **技术栈**: Node.js + Express.js（轻量级Web服务器框架）。
*   **内容形式**:
    *   **人类友好内容**: 一个标准的HTML文件 (`index.html`)，包含产品名称、描述、价格、特点等信息，并带有基本的视觉样式。
    *   **AI友好内容**: 一个结构化的JSON文件 (`product-data.json`)，包含与HTML页面相同的核心产品数据，但以机器易于解析的键值对形式呈现。
*   **核心逻辑**:
    1.  搭建一个Express服务器，监听指定端口（例如3000）。
    2.  定义一个API端点（例如 `/product/1`），模拟某个产品的详情页。
    3.  在该端点内部，服务器会检查传入请求的HTTP `User-Agent` 头。
    4.  **身份识别**:
        *   如果 `User-Agent` 包含特定标识（例如 `"ScrunchAXP-Agent/1.0"`），则认定为AI智能体访问。
        *   否则，认定为人类用户（或普通爬虫）访问。
    5.  **动态内容分发**:
        *   针对AI智能体，服务器将返回预先准备好的 `product-data.json` 内容，并设置响应头 `Content-Type: application/json`。
        *   针对人类用户，服务器将返回 `index.html` 文件的内容，并设置响应头 `Content-Type: text/html`。

## 4. MVP演示与验证

通过 `curl` 工具模拟带有或不带有特定 `User-Agent` 的HTTP请求，我们将能够清晰地观察到服务器根据访问者身份返回了不同格式的内容，从而验证双轨内容分发的核心机制。

## 5. MVP未来拓展（非本次范围）

此MVP仅验证了最基础的内容切换机制。一个完整的 Scrunch AXP 平台将可能包括：

*   **内容结构化工具**: 辅助企业将非结构化内容转化为统一的语义模型（如基于Schema.org）。
*   **API网关**: 统一管理AI智能体的API访问，提供身份验证、限流等功能。
*   **内容渲染引擎**: 更复杂的服务器端渲染（SSR）策略，以确保AI爬虫也能获取到完整内容。
*   **智能体集成**: 与主流AI平台（如ChatGPT Plugins, Google Gemini Extensions）的深度集成。
*   **效果监控与优化**: 跟踪AI解析成功率、内容调用频率等指标，持续优化内容模型。

这个MVP为Scrunch AXP的未来发展奠定了坚实的基础，验证了内容为AI服务，同时不牺牲人类体验的可行性。
