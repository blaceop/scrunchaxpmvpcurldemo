# Scrunch AXP MVP - Bilibili 演示计划

## 1. 项目背景与核心问题

在您提出的“将www.bilibili.com转化为Scrunch AXP结构化数据”的宏伟蓝图指导下，本项目已从一个通用产品模型，**升级为一个针对B站视频页的、高度具体的适配演示**。

核心问题保持不变：如何让AI Agent高效、准确地理解和利用为人类设计的复杂网页内容（如B站视频页）。

## 2. MVP目标

本MVP旨在模拟并验证您方案中的核心环节：**将一个模拟的B站视频页，通过双轨制，同时服务于人类访客和AI Agent**。

*   **人类用户**: 访问`/video/BV1xx411c7mZ`时，获得一个模仿B站风格的、动态渲染的HTML页面。
*   **AI Agent**: 访问同一URL时，获得一个为您方案中定义的、结构化的视频信息JSON。

## 3. MVP技术选型与实现 (v4 - B站场景适配)

*   **技术栈**: Node.js + Express.js + EJS (模板引擎)。
*   **单一数据源**: `video-data.json` 文件，其结构完全遵循您为B站视频内容定义的Schema。
*   **核心逻辑**:
    1.  **B站风格模板**: `views/video.ejs` 文件被创建，用于渲染一个简化的B站视频播放页。
    2.  **B站风格路由**: 服务器的核心路由已变更为 `GET /video/BV1xx411c7mZ`。
    3.  **动态渲染流程**:
        *   当**人类用户**访问时，服务器读取`video-data.json`，并使用EJS模板引擎渲染出`video.ejs`页面。
        *   当**AI Agent**访问时，服务器直接返回`video-data.json`的原始内容。
    4.  **配置与日志**: `config.json`和结构化日志机制继续沿用，为这个B站演示提供可维护性和可观察性。

## 4. MVP演示与验证

演示方式将完全围绕B站场景：

1.  用浏览器访问`/video/BV1xx411c7mZ`，将看到一个模拟的B站页面。
2.  用带特定`User-Agent`的`curl`访问，将得到结构化的视频JSON。
3.  通过修改`video-data.json`中的视频标题或播放量，并刷新浏览器，可以直观地验证“单一数据源”驱动“双轨内容”的强大能力。

## 5. 总结

这次改造使MVP不再是一个抽象的模型，而是您“B站适配方案”的一个**可运行、可感知**的微型实例。它清晰地展示了如何剥离视觉层、提取核心数据，并为AI和人类提供截然不同但同源的内容体验。

## 6. MVP未来拓展（非本次范围）

此MVP为Scrunch AXP的未来发展奠定了坚实的基础，验证了内容为AI服务，同时不牺牲人类体验的可行性。一个完整的 Scrunch AXP 平台将可能包括：

*   **内容结构化工具**: 辅助企业将非结构化内容转化为统一的语义模型（如基于Schema.org）。
*   **API网关**: 统一管理AI智能体的API访问，提供身份验证、限流等功能。
*   **内容渲染引擎**: 更复杂的服务器端渲染（SSR）策略，以确保AI爬虫也能获取到完整内容。
*   **智能体集成**: 与主流AI平台（如ChatGPT Plugins, Google Gemini Extensions）的深度集成。
*   **效果监控与优化**: 跟踪AI解析成功率、内容调用频率等指标，持续优化内容模型。
